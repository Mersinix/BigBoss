BIGBOSSCOFFEE - AUTHENTICATION & ROLE REGISTRATION REFACTOR

IMPORTANT RULES

* Before making any modification, analyze the current project structure completely.
* Reuse existing authentication, approval, user, supplier, coffee, admin, settings, location and modal components whenever possible.
* Do NOT break existing functionality.
* Do NOT change any existing business logic that is not explicitly mentioned below.
* Follow the same architecture, folder structure, coding conventions, UI patterns, routing system, database patterns and state management already used in the project.
* Create only the modifications described in this specification.
* If an existing component already satisfies part of the requirement, extend it instead of creating a duplicate.

====================================================

1. LANDING PAGE LOGIN FLOW
   ====================================================

Current behavior:

* Landing page contains a "Connexion" button.

New behavior:

* Clicking "Connexion" must open a modal.

The modal must display role cards with:

* Icon
* Color
* Title
* Short description

Roles:

1. Café
2. Supplier
3. Delivery
4. Printer
5. Marketing
6. Barista Academy
7. Barista Marketplace

Each role card must have its own visual identity while remaining consistent with the landing page design.

When a role is clicked:

* Redirect user to Authentication page
* Selected role must be passed in route/state/query parameter
* Authentication page must automatically display the correct registration form for that role
