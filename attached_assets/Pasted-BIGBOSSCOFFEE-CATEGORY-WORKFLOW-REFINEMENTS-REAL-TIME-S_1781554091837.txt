====================================================

1. SUPPLIER CATEGORY REQUESTS IMPROVEMENT
   ====================================================

Current behavior:

A supplier can create Category, SubCategory, Brand, Flavor and Size suggestions.

Required change:

When a supplier creates a Category suggestion with:

status = PENDING

that Category must immediately become available in the supplier's own Category Requests section when creating:

* SubCategory
* Brand
* Flavor
* Size

The supplier should NOT be forced to wait for admin approval before creating related hierarchy elements.

Example:

Supplier creates:

Coffee Beans
(status = PENDING)

Immediately afterwards he must be able to create:

Premium Coffee Beans
(parent = Coffee Beans)

even though Coffee Beans is still Pending.

Result:

The supplier can prepare the entire hierarchy:

Category
→ SubCategory
→ Brand
→ Flavor
→ Size

before any admin approval.

All newly created records remain:

status = PENDING

until approved by admin.

2. SUPPLIER MY CATEGORIES FILTERING
===================================

Current behavior should be adjusted.

Inside:

Supplier

* Categories linked to products owned/used by supplier
* SubCategories linked to products owned/used by supplier

Hide empty categories.

Hide empty subcategories.

====================================================
3. PRODUCTS DEPEND ON CATEGORY TAB SELECTION
============================================

The selected Category and SubCategory from:

Supplier
→ Categories
→ My Categories

must become the single source of truth for product visibility.

The Products page must use the Category/SubCategory selected in My Categories.

This dependency applies to:

Supplier
→ Products
→ Admin Products

and

Supplier
→ Products
→ My Products

---

## INITIAL STATE

If no Category has been selected in:

Categories
→ My Categories

then:

* No Category should appear in Product filters.
* No SubCategory should appear in Product filters.
* No products should be displayed.

This applies to both:

* Admin Products
* My Products

---

## CATEGORY / SUBCATEGORY DEPENDENCY

When a Category is selected from:

Categories
→ My Categories

the Products page should automatically use that selection.

When a SubCategory is selected from:

Categories
→ My Categories

the Products page should automatically use that selection.

Products displayed inside:

* Admin Products
* My Products

must be filtered using the currently selected Category and SubCategory from My Categories.

Rules:

IF no Category selected:

* No Category filter displayed.
* No SubCategory filter displayed.
* No products displayed.

IF Category selected but no SubCategory selected:

* No products displayed.

IF Category selected and SubCategory selected:

* Display only products belonging to that selected SubCategory.

This creates a centralized navigation flow:

Categories > My Categories
→ Select Category
→ Select SubCategory
→ Products > Admin Products
→ Products > My Products

The Products page should not maintain a separate Category/SubCategory selection state.

It must reuse the active selection coming from My Categories.
