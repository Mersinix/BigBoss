IMPLEMENTATION REQUIREMENTS

1. LANDING PAGE NAVBAR

Locate the location button in the landing page navbar.

When the user clicks the location button on the landing page:

* Open the existing LocationPickerModal.

2. SHARED LOCATION STATE

The landing page and shop must always use the same saved location.

When a user selects a location from the landing page:

* Save it using the same endpoint currently used by the marketplace:
  PATCH /api/auth/me/location

* Use the same payload structure already implemented.

* Invalidate the same React Query caches currently invalidated in the marketplace implementation.

* Immediately refresh the authenticated user data.

3. SYNCHRONIZATION

After saving a location from the landing page:

* Navigating to /shop must automatically show the same location.
* Opening the LocationPickerModal from the SHOP navbar must display the previously selected address.
* The navbar location label must update instantly without requiring a page refresh.
* Reloading the application must keep the selected location.

4. GUEST USERS

For non-authenticated users:

* Keep the same UX as much as possible.
* Persist the selected location locally (localStorage/session mechanism already used in the project if available).
* When the user later signs in, automatically synchronize that location with the account if the application already has a synchronization mechanism.

5. UI CONSISTENCY

The landing page location button must display the same formatted label logic used in the marketplace:

Example:
"Le Bardo, Tunis" → "Le Bardo"
