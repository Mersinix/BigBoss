# 1. Complete "Specific Products" and "Specific Categories" Assignment

Currently, in the Create/Edit Promotion modal, there are three options under **Applies To**:

* All my products ✅ (works correctly)
* Specific products
* Specific categories

For the last two options, the modal displays:

> "You can configure specific products after saving this promotion."

or

> "You can configure specific categories after saving this promotion."

However, after saving the promotion there is currently **no interface** where the supplier can actually configure those assignments.

This feature needs to be completed.

---

## Promotion Details Page

After a promotion is created, the supplier should be able to open its details (or edit page) and manage assignments.

Create two new management sections.

---

## Products Assignment

If the promotion applies to **Specific Products**, provide a complete product assignment interface.

The supplier should be able to:

* Browse only **their own supplier products**
* Search products
* Filter by category
* Select multiple products
* Select all
* Unselect all
* Remove products
* Add products later
* See how many products are currently assigned

Each assigned product should display:

* Product image
* Product name
* Category
* Current selling price
* Current stock
* Promotion status

Changes should save without requiring recreation of the promotion.

---

## Categories Assignment

If the promotion applies to **Specific Categories**, provide a category assignment interface.

The supplier should be able to:

* View only categories containing their own products
* Search categories
* Select multiple categories
* Remove categories
* See number of assigned products inside each category

When a category is assigned:

All current products inside that category automatically receive the promotion.

Future products added by the supplier into the same category should automatically inherit the promotion while it remains active.

---

## Database

Create proper relationships instead of storing comma-separated IDs.

Example:

PromotionCategories

* promotionId

Use existing project architecture if equivalent mapping tables already exist.

Examples:

⚡ 20% OFF

🔥 Buy 10 Get 1

🚚 Free Shipping

🎁 Free Gift

💰 Save 50 DT

The badge should be visible directly in the product grid before opening the product.

---

## Badge Resolution

The badge logic must correctly support all promotion scopes:

### Applies To

* All my products
* Specific products
* Specific categories

For every product card, determine whether an active promotion applies.

The badge should appear regardless of how the promotion was assigned.

---

## Priority

If multiple promotions apply to the same product:

Display only the highest-priority promotion badge according to the existing promotion engine rules.

Do not display multiple badges unless the current UI already supports it elegantly.

## Product Card Information

When a promotion exists, the card should display:

* Promotion badge
* Original price (strikethrough when applicable)
* Discounted price
* Small "Promotion" indicator

Only if the promotion type affects price.

For non-price promotions (Free Gift, Free Shipping, Buy X Get Y), display only the appropriate promotional badge.
