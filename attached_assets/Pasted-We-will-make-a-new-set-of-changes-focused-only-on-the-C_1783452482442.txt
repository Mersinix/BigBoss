We will make a new set of changes focused only on the Coffee Owner Store experience, the Supplier Store configuration, and the Admin Store management.

Important requirements before starting:

Do not change the existing design language.
Do not change existing business logic outside the scope below.
Do not remove or refactor unrelated features.
Preserve all existing APIs and behaviors unless they must be extended for these requirements.
Check existing Store, Reviews, Favorites, Product Details, Product Cards and synchronization logic before making any changes to avoid duplicate implementations.
Keep everything fully synchronized in real time (admin ↔ supplier ↔ coffee owner) exactly like the rest of the project. Any create/update/delete/approve/order/visibility action must immediately update the UI without requiring a page refresh.
1. Coffee Owner → Home → Stores section

The current Store cards are working correctly.

Update only their presentation.

1.1 Horizontal scrolling

Instead of displaying all stores in a normal grid:

Display the first row as a horizontal scroll carousel.
The user should be able to scroll horizontally through the stores.
Keep the existing Store card design.
1.2 Move the "See More / Show Less" button

Move the button to the same horizontal line as the section title.

Example:

Marques populaires                         See More

When expanded:

Marques populaires                         Show Less

The button should no longer appear at the bottom of the component.

1.3 Expanded mode

When the user clicks See More:

expand the component
display all stores
preserve all existing filters
preserve all existing sorting

When clicking Show Less:

return to the horizontal scrolling view.
1.4 Empty stores

If a store has no available products, it must never be displayed.

This rule must continue respecting:

supplier visibility
admin approval
supplier frozen categories
product availability
existing product visibility logic

Do not break the current filtering behavior.
