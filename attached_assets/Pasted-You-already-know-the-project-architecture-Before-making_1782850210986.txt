# 5. Improve the Pending Approval experience

Review the registration and login experience for account types that require administrator approval.

The existing approval workflow must remain unchanged.

Only improve the user experience.

### After registration

When a user successfully creates an account requiring administrator approval:

Display a clear confirmation message explaining that:

* the account has been created successfully;
* the account is currently waiting for administrator approval;
* the user cannot access the application until approval;
* the user will receive a notification (email or any configured notification channel) once the account has been approved.

The message should make it obvious that registration succeeded and that waiting for approval is expected.

---

### During login

If a user whose account is still pending approval attempts to log in:

* do not allow access to the application;
* display a friendly message explaining that the account is still waiting for administrator approval;
* explain that access will become available once the administrator approves the account;
* after the user acknowledges the message, redirect them back to the Landing Page.

The experience should clearly indicate that this is not a login error but the normal approval process.

Approved users must continue logging in exactly as they do today.
